# Builder API

The builder API lets you create themes from minimal input. Missing values are generated automatically.

## `createTheme()`

Returns a chainable `ThemeBuilder` that accumulates configuration and produces a `Theme` via `.build()`.

```typescript
import { createTheme } from "themex"

const theme = createTheme()
  .bg("#2E3440")
  .primary("#EBCB8B")
  .build()
```

### Builder Methods

| Method                  | Description                                      |
|-------------------------|--------------------------------------------------|
| `.bg(color)`            | Set background color (maps to palette `base`)    |
| `.fg(color)`            | Set foreground color (maps to palette `text`)     |
| `.primary(color)`       | Set primary accent color                          |
| `.accent(color)`        | Alias for `.primary()`                            |
| `.dark()`               | Force dark mode                                   |
| `.light()`              | Force light mode                                  |
| `.color(name, value)`   | Set any palette color by name                     |
| `.palette(p)`           | Set full palette at once                          |
| `.preset(name)`         | Load a built-in palette by name                   |
| `.build()`              | Derive the final `Theme`                          |

### How the Builder Fills Gaps

1. **Dark/light detection**: If not explicit (`.dark()` or `.light()`), inferred from background luminance
2. **Surface ramp**: If `.bg()` is set without a preset, the full ramp is generated:
   - `crust`: slightly darker (dark) or lighter (light) than base
   - `surface`: slightly lighter (dark) or darker (light) than base
   - `overlay`, `subtext`, `text`: progressively farther from base
3. **Accent hues**: If `.primary()` is set without a preset, all 8 accents are generated by:
   - Detecting which hue slot the primary belongs to (e.g., `#EBCB8B` -> yellow)
   - Rotating around the hue wheel at fixed intervals, preserving saturation and lightness
   - Keeping the original primary color at its natural slot
4. **Preset override**: If `.primary()` is used with a preset, the primary is placed in its natural hue slot within the preset palette
5. **Final derivation**: `deriveTheme()` produces all 19 semantic tokens from the assembled palette

## Examples

### Minimal: Just Background

```typescript
const theme = createTheme().bg("#2E3440").build()
```

Gets you: auto-detected dark mode, generated surface ramp, default Nord-inspired accents, all 19 semantic tokens.

### Two Colors: Background + Primary

```typescript
const theme = createTheme()
  .bg("#1A1B26")
  .primary("#E0AF68")
  .build()
```

Gets you: full surface ramp from background, 8 accent hues generated from primary, yellow used as `$primary` token.

### Three Colors: Full Control

```typescript
const theme = createTheme()
  .bg("#2E3440")
  .fg("#ECEFF4")
  .primary("#EBCB8B")
  .build()
```

### Preset with Customization

```typescript
// Override just the primary accent of Nord
const theme = createTheme()
  .preset("nord")
  .primary("#A3BE8C")  // green instead of yellow
  .build()
```

### Individual Color Overrides

```typescript
const theme = createTheme()
  .preset("catppuccin-mocha")
  .color("red", "#FF0000")
  .color("surface", "#222244")
  .build()
```

### Explicit Mode Override

```typescript
// Force light mode even with a dark-ish background
const theme = createTheme()
  .bg("#3A3A3A")
  .light()
  .build()
```

### Full Palette

```typescript
import type { ThemePalette } from "themex"

const myPalette: ThemePalette = {
  name: "custom", dark: true,
  crust: "#11111B", base: "#1E1E2E", surface: "#313244",
  overlay: "#6C7086", subtext: "#A6ADC8", text: "#CDD6F4",
  red: "#F38BA8", orange: "#FAB387", yellow: "#F9E2AF",
  green: "#A6E3A1", teal: "#94E2D5", blue: "#89B4FA",
  purple: "#CBA6F7", pink: "#F5C2E7",
}

const theme = createTheme().palette(myPalette).build()
```

## Convenience Functions

### `quickTheme(primaryOrHex, mode?)`

One-liner theme creation from a primary color.

```typescript
import { quickTheme } from "themex"

// From hex
const theme = quickTheme("#EBCB8B", "dark")

// From color name
const theme2 = quickTheme("blue")
const theme3 = quickTheme("teal", "light")
```

Supported color names: `red`, `orange`, `yellow`, `green`, `teal`, `cyan`, `blue`, `purple`, `pink`, `magenta`, `white`.

**Signature:**
```typescript
function quickTheme(primaryOrHex: string, mode?: "dark" | "light"): Theme
```

### `presetTheme(name)`

One-liner theme creation from a built-in preset.

```typescript
import { presetTheme } from "themex"

const theme = presetTheme("catppuccin-mocha")
const theme2 = presetTheme("nord")
const theme3 = presetTheme("dracula")
```

Equivalent to `createTheme().preset(name).build()`.

**Signature:**
```typescript
function presetTheme(name: string): Theme
```

### `generateTheme(primary, dark)`

Generate an ANSI 16 theme from a primary color name. These themes use ANSI color names (not hex values) and work in any terminal.

```typescript
import { generateTheme } from "themex"

const theme = generateTheme("yellow", true)   // dark theme, yellow primary
const theme2 = generateTheme("cyan", false)    // light theme, cyan primary
```

Supported primaries: `yellow`, `cyan`, `magenta`, `green`, `red`, `blue`, `white`.

**Signature:**
```typescript
function generateTheme(primary: AnsiPrimary, dark: boolean): Theme
```
