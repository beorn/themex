# Creating Themes

## The Two-Layer System

swatch uses a two-layer architecture:

1. **ColorPalette** (Layer 1) -- 22 terminal colors (16 ANSI + 6 special)
2. **Theme** (Layer 2) -- 33 semantic tokens derived via `deriveTheme()`

All theme creation flows through this pipeline: **Input -> ColorPalette -> `deriveTheme()` -> Theme**.

## ColorPalette

A `ColorPalette` has 22 colors in the standard terminal format that every modern emulator uses (Ghostty, Kitty, Alacritty, iTerm2, WezTerm):

- **16 ANSI colors**: `black`, `red`, `green`, `yellow`, `blue`, `magenta`, `cyan`, `white` + bright variants
- **6 special colors**: `foreground`, `background`, `cursorColor`, `cursorText`, `selectionBackground`, `selectionForeground`

You rarely fill all 22 manually -- the builder API and generator functions handle this.

## Using the Builder API

The builder lets you create themes from minimal input. It fills in missing values automatically.

### From Just a Background Color

```typescript
import { createTheme } from "swatch"

const theme = createTheme().bg("#2E3440").build()
```

The builder:

1. Detects dark/light from background luminance
2. Generates all 22 palette colors via `fromColors()`
3. Derives all 33 semantic tokens via `deriveTheme()`

### From Background + Primary

```typescript
const theme = createTheme().bg("#2E3440").primary("#EBCB8B").build()
```

The primary color is assigned to its natural hue slot (yellow in this case), and remaining accent hues are generated by rotating around the hue wheel.

### From Three Colors

```typescript
const theme = createTheme().bg("#2E3440").fg("#ECEFF4").primary("#EBCB8B").build()
```

### Preset with Override

```typescript
const theme = createTheme()
  .preset("nord")
  .primary("#A3BE8C") // change primary to green
  .build()
```

### Individual Color Overrides

```typescript
const theme = createTheme().preset("catppuccin-mocha").color("red", "#FF0000").color("background", "#222244").build()
```

### Explicit Dark/Light Mode

```typescript
const darkTheme = createTheme().primary("#EBCB8B").dark().build()

const lightTheme = createTheme().primary("#1976D2").light().build()
```

## Generator Functions

For programmatic use without the builder:

```typescript
import { fromColors, fromPreset, fromBase16, deriveTheme } from "swatch"

// From minimal colors -> ColorPalette -> Theme
const palette = fromColors({ background: "#2E3440", primary: "#EBCB8B" })
const theme = deriveTheme(palette)

// From a built-in preset name
const nordPalette = fromPreset("nord")

// From Base16 YAML
const base16Palette = fromBase16(yamlString)
```

## Validating Palettes

Check that a palette has all required fields and reasonable contrast:

```typescript
import { validateColorPalette } from "swatch"

const result = validateColorPalette(myPalette)

if (!result.valid) {
  console.error("Errors:", result.errors)
}
if (result.warnings.length > 0) {
  console.warn("Warnings:", result.warnings)
}
```

## Global Theme State

For render pipelines where React context is not accessible, swatch provides optional global state:

```typescript
import { setActiveTheme, getActiveTheme } from "swatch"

// Set during render setup
setActiveTheme(theme)

// Read during rendering
const current = getActiveTheme()
```

For per-subtree overrides (like CSS custom properties cascading):

```typescript
import { pushContextTheme, popContextTheme } from "swatch"

pushContextTheme(dialogTheme) // children see dialogTheme
// ... render children ...
popContextTheme() // restore parent theme
```

::: tip
Global state is optional. For most applications, pass `Theme` objects explicitly to `resolveThemeColor()`.
:::
